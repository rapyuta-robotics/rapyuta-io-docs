Welcome to RIO Client’s documentation!¶

RIO Client is a python interface for the service provided by Rapyuta IO. It allows developers to write software that makes use of services provided by Rapyuta IO.

rapyuta_io package¶

rapyuta_io.client module¶

class rapyuta_io.client.Client¶

Client class provide access to device and package class. select_devices and get_package methods can be used to get instance of selected devices and package.

Get instance of the package identified by the service_id and plan_id.

Parameters:
  • package_id (string) – Package Id
  • plan_id (string) – Plan Id
  • auth_token (string) – Authorization token
Returns:

Instance of Package class.
select_devices(query, auth_token=None)¶

Select the devices based on labels and telemetry data.

Parameters:
  • query (string) – Query is a comma separated set of sub-queries. Each subquery can select the device based on label or telemetry data. Label based sub-query will search for the devices based on the device labels for e.g. ‘label.camera=true’ will search for the devices having label ‘camera:true’. Telemetry based sub-query will search for the devices based on the telemetry data for e.g. ‘measurement.battery>20’ will search for the devices having battery greater than 20%. Finally the method will select the devices which meet the criteria of all the sub-queries.
  • auth_token (string) – Authorization token to validate the request.
Returns:

list of instances of the selected devices.

rapyuta_io.clients.catalog module¶

class rapyuta_io.clients.catalog.Config(package, *args, **kargs)¶

Config class contain configuration of the components of package. You need to provide parameters for each component before you deploy the package. The required parameters for each component are present inside the component_parameters attribute of config class. component_parameters is a python dictionary in which keys are component names and value is python dictionary of the required parameters of the corresponding component.

In the component name all hyphens(‘-‘) are replaced by underscores(‘_’)

class rapyuta_io.clients.catalog.Package(*args, **kargs)¶

Package class represents a service package. It contains methods to provision, deprovision an instance of the package on cloud or on device. It has other utility methods also.

Variables:
  • name – Package name.
  • package_id – Id of the package.
  • plan_id – Id of the selected plan.
  • bindable – Boolean representing if the package is bindable.
  • description – Description of the package
  • free – Boolean representing if the selected plan of the package is free.
  • config – Instance 0f Config class.
  • metadata – Python dictionary containing metadata of all the components of the package.
